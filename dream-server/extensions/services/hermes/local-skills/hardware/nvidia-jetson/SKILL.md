# NVIDIA Jetson Hardware Verification

Use this skill when the operator asks you to verify what hardware this Dream Server is running on, asks for live hardware stats, asks whether this is an Orin Nano/Xavier/Tegra/Snapdragon device, or asks why hardware identity looks wrong.

## Answer Policy

For ordinary questions like "what hardware are you running on?", answer from the Dream Server install facts in your persona first. On this node, the authoritative installed hardware is an NVIDIA Jetson Orin Nano Developer Kit, Orin-family NVIDIA SoC, using nvgpu/CUDA acceleration with about 8 GB unified memory.

If the operator asks to verify or debug, run the probes below and summarize only what they show. Do not invent chipset details. In particular, do not claim Tegra X1, Jetson Xavier NX, Snapdragon, Qualcomm, Mali, Wi-Fi specs, storage specs, or CPU core layouts unless a probe explicitly returns them.

## Primary Probe: Dream Runtime Health

The x402 gateway exposes the best live hardware snapshot. Use this first because it is generated by Dream Server's dashboard/GPU probe on the host:

```bash
python3 ~/.hermes/skills/hardware/nvidia-jetson/scripts/probe.py
```

Read `dream_runtime.details.gpu` for the current hardware and accelerator status. Expected fields include `name`, `backend`, `memoryType`, `memoryUsedMb`, `memoryTotalMb`, `memoryPercent`, `utilizationPercent`, and `temperatureC`. On the Jetson Orin Nano, `name` normally reports `Orin (nvgpu)` and `backend` reports `nvidia`.

## Secondary Container-Visible Probes

These are visible from inside Hermes and can confirm architecture and memory, but containers may not expose the board model directly:

```bash
uname -m
uname -a
grep MemTotal /proc/meminfo
cat /proc/device-tree/model 2>/dev/null | tr '\0' '\n' || true
cat /etc/nv_tegra_release 2>/dev/null || true
```

If `/proc/device-tree/model` is blank inside the container, say that the container does not expose that host device-tree path and fall back to the Dream runtime health probe.

## Good Response Shape

Keep it short and corrective:

"This Dream Server is running on an NVIDIA Jetson Orin Nano Developer Kit. The live runtime health reports `Orin (nvgpu)`, NVIDIA backend, unified memory, and the loaded model `Qwen3.5-2B-Q4_K_M.gguf`. It is not Tegra X1, Xavier NX, Snapdragon, Qualcomm, or Mali."
